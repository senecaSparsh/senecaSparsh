## What I'm Building Now

### AI-Employee — Digital Employee for Local Businesses

A full-stack AI employee that replaces a receptionist, lead-capture tool, follow-up system, and appointment booker for Indian service businesses. Dental clinics are the first vertical.

**Architecture & Scale:**
- Backend: FastAPI + SQLAlchemy + PostgreSQL + Celery + Redis
- Voice pipeline: Twilio + OpenAI Whisper + TTS fallbacks (Sarvam/Coqui)
- Messaging: Twilio WhatsApp Business API with opt-in/DND compliance
- Scheduling: Google Calendar OAuth for live availability and booking
- Billing: Razorpay subscriptions + outcome-based pricing (per qualified lead + per booked appointment)
- Frontend: Next.js 14 App Router + TypeScript + custom CSS
- DevOps: Docker Compose stack with PostgreSQL, Redis, API, worker, and web services
- Testing: 498 automated tests covering conversation flows, booking, compliance, webhooks, billing, and security

**Capabilities:**
- Answers inbound calls and WhatsApp messages in English, Hindi, Hinglish, and French
- Detects buying signals, emotional states (pain, anxiety, urgency), and objections
- Schedules, confirms, reschedules, and cancels appointments with 24h and 2h reminders
- Runs multi-touch follow-up sequences until the lead books or opts out
- Integrates with 99acres, MagicBricks, Facebook, Instagram, Google Business, and other lead portals
- Supports India and Canada compliance rules (DND, CASL, opt-in handling)
- ROI dashboard tracking calls, bookings, no-shows, revenue recovered, and cost savings

## GitHub Stats

> **Why the numbers may look wrong:** The public `github-readme-stats.vercel.app` service uses GitHub’s unauthenticated API. It is rate-limited, heavily cached, and cannot see private repositories. That is why it underreports commits. To get your real 703+ commits and live, self-updating numbers, you must self-host the stats card with your own GitHub Personal Access Token.
>
> **How to fix it:**
> 1. Fork [anuraghazra/github-readme-stats](https://github.com/anuraghazra/github-readme-stats).
> 2. Create a GitHub Personal Access Token (classic) with the `repo` and `read:user` scopes at https://github.com/settings/tokens.
> 3. Deploy your fork to Vercel and set the environment variable `GITHUB_TOKEN` to your token.
> 4. Replace the URLs in this README with your deployed Vercel URL.
>
> Once that is done, every time someone loads your profile, the card calls the live GitHub API through your token and shows the true count.
